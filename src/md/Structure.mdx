### 비선형 구조란?
비선형 구조(NonLinear)는 `일대일`로 연결되어 있지 않고, `일대다` 또는 `다대다`의 관계를 맺는 구조를 형성하는 구조이다.

### 트리란?  
각각의 자료들을 계층적으로 서로 연결한 자료구조이다.
`포함 관계` 혹은 `상 하위 관계`를 표현할 때 사용된다.

가장 상위에 있는 노드를 `최상위(root)` 노드라고 하며
자식 노드와 연결된 간선의 개수가  
 `하나도 없는` 노드를 `단말(Leaf)` 노드라고 한다.

트리의 용어를 더 자세히 알아보자.

### 👉 형제 노드
같은 부모를 가지는 노드를 말한다.   
### 👉 노드의 깊이
`root`노드에서 어떤 노드에 도달하기 위해 거쳐야 하는 간선의 수를 뜻한다.    
### 👉 노드의 레벨 
`node`의 `level`은 트리의 특정 깊이를 가지는 노드의 집합을 말한다.  
위의 트리를 보면 각 `level` 마다 노드의 집합이 달라지는 것을 알 수 있다 
### 👉 노드의 차수 
노드의 차수 `degree`는 각 노드가 지닌 가지의 수를 뜻한다  
### 👉 트리의 높이
트리의 높이 `height`는 루트 노드에서 가장 깊숙히 있는 노드의 깊이이다.  
### 👉 트리의 차수
트리의 차수 `degree of tree`는 트리의 `최대 차수`이다.    

`트리의 차수`는 `최대 차수`를 이용해 구할 수 있다.  예를 들면  
```text
     10
    /  \
   4   14
  / \
 2   6
```
만약 위와 같은 트리가 있다면 차수는 `2`이다.  
만약 하나의 노드가 자식이 `3`개 있다면 최대 차수가 `3`이 되므로 그 트리는 차수가 `3`이 된다.
### 이진 트리

<img src="https://user-images.githubusercontent.com/45661217/146921745-96f811bd-7ee1-4201-95b7-ecc674a6fadb.png" width="70%"></img>

다음 그림과 같이 모든 노드의 차수가 `2 이하인 트리`를 `이진 트리` 라고 한다.  

### 이진 트리의 성질  
이진 트리에서 노드의 개수를 `n`개로 정의하였을 때, 간선의 개수는 항상 `n-1`개를 가진다.  
또, 높이가 `h`일때 노드의 최소 개수는 `h`개 이며, 최대 개수는 `2^h-1`개로 정의 할 수 있다.  

### 이진 트리의 종류

#### 포화 이진 트리

모든 레벨에 노드가 완전히 찬 트리를 `포화 이진 트리`라고 한다.  
포화 이진 트리에서 최대 레벨 수를 `k`라고 한다면 전체 노드의 수는 `2^k-1`로 쉽게 구할 수 있다
```text
// 포화 이진 트리

      10   
    /    \
   4      14
  / \    /  \
 2   6  5    7
```

#### 완전 이진 트리

높이가 `h`인 트리에서 레벨 `h-1`까지 모든 노드가 채워져 있고 마지막 레벨에서는 노드가 왼쪽에서 오른쪽으로
저장된 구조를 `완전 이진 트리`라고 한다.  

```text
// 완전 이진 트리

      10   
    /    \
   4      14
  / \      
 2   6      
```
> 모든 포화 이진트리는 완전 이진 트리이다

#### 편향 이진 트리

각 노드가 왼쪽 혹은 오른쪽으로 치우쳐 있는 구조를 `편향 이진 트리`라고 한다.  
```text
// 편향 이진 트리

     10
    /  
   4   
  / 
 2   
```

### 이진 트리의 순회

이진 트리를 출력할 때나 값을 읽을 때 한번씩 차례대로 방문해야한다.  
이진 트리를 방문하는 방법으로는 세가지 방법이 있다.

```text
      10   
    /    \
   4      14
  / \    /  \
 2   6  5    7
```
#### 전위 순회

`root`노드 -> 왼쪽 서브 트리 -> 오른쪽 서브 트리 순서  
10 -> 4 -> 2 -> 6 -> 14 -> 5 -> 7  

#### 중위 순회

왼쪽 서브 트리 -> `root`노드 -> 오른쪽 서브 트리 순서  
2 -> 4 -> 6 -> 10 -> 5 -> 14 -> 7

#### 후위 순회

왼쪽 서브 트리 -> 오른쪽 서브 트리 -> `root`노드 순서  
2 -> 6 -> 4 -> 5 -> 7 -> 14 -> 10

### 트리의 구현

#### 배열을 이용한 트리의 구현

```c
#include<stdio.h>

// 전위 순회
// 전달인자로 tree, index, 트리의 size를 갖는다
void preorder( char *t, int n, int size ){
	if(n>size||t[n]==0) return ;
	printf("%c ", t[n]);
	preorder(t, 2*n, size);
	preorder(t, 2*n+1, size);
}

int main(){
	//비어있는 곳은 0이라는 숫자를 넣음
	char tree[]= { 0,'A', 'B', 'C', 'D', 'E', 'F', 'G', 0, 0,'H', 'I', 0, 'J', 'K', 'L'};
	int size=16;
	printf("\n<<전위 탐색>>\n");
	preorder( tree, 1, size );
	printf("\n<<중위 탐색>>\n");
	inorder( tree, 1, size );
	printf("\n<<후위 탐색>>\n");
    postorder( tree, 1, size );
}
```
배열을 이용한 트리의 구현은 간단하다

```text
      1  
    /   \
   2     3
  / \   /  \
 4   5 6    7
```
트리의 `index`가 이렇게 있다면 왼쪽 아래 노드의 `index`는 `index*2` 오른쪽 아래는 `index*2+1`로 접근할 수 있다.    
위의 코드는 이 규칙을 이용해 `재귀 함수`로 트리 배열을 출력한 모습이다.   

#### 연결 리스트를 활용한 트리의 구현

하나의 노드는 왼쪽 서브 트리의 주소를 갖는 `left`   
오른쪽 서브 트리의 주소를 갖는 `right`가 존재하고 `data`로 노드의 값을 갖는다


```c
typedef struct treeNode{
	char data;
	struct treeNode *left;
	struct treeNode *right;
} treeNode;
```
노드의 구조는 이런 형식으로 구성된다

<h2 id="graph">그래프</h2>

**간선(Edge)** **정점(Vertax)**

### 그래프란? 

그래프는 `정점`과 `간선`으로 이루어진 자료구조이다    
그래프 에서 자료가 저장되어 있는 각 노드를 `정점(vertax)`이라고 한다.  
이러한 정점을 연결하는 링크를 `간선(edge)`이라고 한다.  

> `트리`는 계층형 구조를 표현 하는 자료구조라면 `그래프`는 트리로 표현하기 어려운 관계를 나타낼 수 있다.

다음 그림에서 정점의 개수는 7개 이며, 간선의 개수도 7개이다.

### 그래프의 종류

![그래프의 종류](https://user-images.githubusercontent.com/45661217/147653444-73a741ad-e9b4-45e6-8498-6eb783f4830e.png)  
그래프는 크게 무방향 그래프, 방향 그래프, 완전 그래프, 가중치 그래프로 나뉜다.

#### 👉 무방향 그래프

정점과 정점을 잇는 간선이 방향을 가지지 않는 그래프를 말한다.

#### 👉 방향 그래프

무방향 그래프와 반대로 그래프를 구성하고 있는 정점 사이에 간선이 방향성을 가지는 그래프를 방향 그래프라고 한다.

#### 👉 완전 그래프

그래프 내의 모든 정점이 서로 직접 연결되어 있는 형태의 그래프이다.

#### 👉 가중치 그래프

간선에 비용이나 길이와 같은 가중치를 수치로 부여하여 만든 그래프이다.

### 그래프의 탐색

그래프에서 연산을 수행하기 위해선 자료의 모든 정점을 한번씩 돌아야 하는데 이것을 그래프의 탐색 혹은 그래프의 순회라고 한다.
